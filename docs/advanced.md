## 本地检索模式

本地模式把嵌入和重排模型运行在你的电脑上。它适合检索过程不能联网，或者希望完全自行控制检索模型的场景。

选择本地模式后：

- 首次需要下载约 1.2 GB 模型；
- 建立语义索引和第一次检索通常更慢；
- 检索时会占用更多内存；
- 不再需要把检索文本发送给硅基流动；
- Agent 客户端本身是否联网，仍由对应客户端和模型决定。

### 什么时候值得使用

- 单位或课题要求检索文本不得发送给外部服务；
- 网络环境长期不稳定；
- 电脑内存和性能足够；
- 你愿意等待本地模型准备和索引。

### 切换前要知道

SiliconFlow 云端检索和本地模式生成的向量不兼容。更换检索引擎或模型后，PaperPiggy 会重新建立语义索引。不要在大型文库中频繁切换。

## 文件夹模式

PaperPiggy 当前主要基于与 Zotero 的联动开发。后续会逐步增加对其他文献管理工具的支持；在此之前，文件夹模式是没有 Zotero 时的备用方案，而不是推荐主流程。

文件夹模式支持 PDF、EPUB、DOCX、Markdown 和 TXT，不支持 HTML。

与 Zotero 模式相比，它的限制主要是：

- 文件本身通常没有完整的结构化题录；
- 需要 AI 从全文中抽取题名、作者、年份和期刊；
- 抽取结果需要人工核对；
- 没有 Zotero 收藏夹和条目关系；
- 文件移动、改名和同步删除需要更谨慎。

适合以下情况：

- 尚未使用任何文献管理软件；
- 手上只有一批独立全文文件；
- 临时建立一个与主 Zotero 文库分开的资料库。

如果已经使用 Zotero，优先把全文附到 Zotero 条目上，再让 PaperPiggy 更新知识库。

## 应用内对话

PaperPiggy 自带“对话”页，可以填写大语言模型 API Key 后直接向知识库提问。它适合：

- 临时问一个简单问题；
- 快速查看某个主题的库内回答；
- 不想启动外部 Agent 客户端时做轻量查询。

但它不适合承担跨会话的长期研究流程。与 Agent 相比，应用内对话不擅长连续执行复杂工作流、维护项目记忆、管理交付物和系统性养护综述库。

因此，日常主线仍推荐 Agent；应用内对话作为轻量补充。

## 手动使用“文献”页

“文献”页提供两类检索：

- **题录检索**：按题名、作者、期刊、年份、DOI 或 ISBN 查找；
- **全文与相关内容**：按研究问题检索相关文献和原文片段。

它适合快速浏览、筛选和人工抽查。需要系统性研究时，把任务交给 Agent 更省力；Agent 会先做发现型检索，再对关键来源定向深读。

## 手动管理深索

文献页可以按“已深索 / 未深索”和“检索摘要有效 / 异常 / 缺失”筛选，并手工选择一批文献处理。

适合手动操作的情况：

- 你已经明确知道今天要读哪几篇；
- 某一篇文献需要立即全文可检索；
- 需要观察具体失败项；
- 想暂停或继续一个已有队列。

大型文库仍建议分批处理，或在电脑夜间空闲时运行。扫描版 PDF 会自动本地 OCR，因此耗时通常更长。

## 检索摘要的三种生成方式

在“设置 → 建库 → 检索摘要”中可以选择：

| 方式 | 适合情况 |
|---|---|
| 交给 Agent 生成 | 推荐。Agent 读取正文、写摘要并重嵌入，质量更可控 |
| PaperPiggy 自动生成 | 在应用内点深索后自动生成。下方直接选择复用 SiliconFlow Key，或使用其他 AI 厂商 |
| 暂不生成 | 深索仍正常完成，以后可随时补生成 |

选择“PaperPiggy 自动生成”后：

- **复用检索引擎的 SiliconFlow Key**：使用固定的简单模型，该模型当前免费；
- **使用其他 AI 厂商**：可选 DeepSeek、Kimi、智谱、OpenAI 或自定义兼容服务，并自行指定模型，通常按量收费。

这只是检索摘要的生成来源，不会改变检索引擎。检索摘要是深索后的可选增强；没有摘要不影响读取已经深索的全文。

## 清空并重建索引

“清空并从头重建索引”不是普通刷新按钮。只有在索引长期异常、重复文献需要重新处理，或版本说明明确要求时才使用。

普通新增或修改文献，点击“手动更新知识库”即可。清空重建前务必先备份；它会重新生成全文、切块、向量、BM25 和深索进度，但会保留综述库、分类、检索摘要、设置和 Agent 工作区。
