# 接入 AI Agent

## 什么时候应该使用 Agent

| 内置对话 | AI Agent |
|---|---|
| 随手问一个问题 | 系统研究一个主题 |
| 一次读取少量来源 | 多轮检索、深读和核验 |
| 答案默认不跨会话积累 | 维护交付物、综述和项目记忆 |
| 手动保存答案 | 可以执行完整研究工作流 |

如果你的任务是“围绕一个主题写综述”“逐项核对引文”“长期维护知识库”，Agent 更合适。

## 支持哪些助手

PaperPiggy 可以接入支持 MCP 的 AI 助手，例如 Claude Code、Codex，以及其他允许配置本地 MCP 服务的工具。

接入命令包含当前电脑的真实安装路径，因此不要照抄别人教程里的固定命令，应当使用你自己 PaperPiggy 页面生成的配置。

## 接入步骤

1. 打开“🤖 Agent”。
2. 点击“Agent 使用指引”。
3. 选择你正在使用的 AI 助手。
4. 复制页面生成的命令或配置。
5. 按页面提示添加。
6. 关闭旧会话，重新打开一个新会话。
7. 确认 AI 助手能看到 `localkb`。

配置通常在新会话中才会加载。如果当前会话没有看到知识库，先重新打开会话，不要反复改同一份配置。

## 第一次只读测试

新会话中先发送：

```text
请检查 PaperPiggy 是否连接成功，然后列出我最近入库的 5 篇文献。
这一步只读，不要修改任何内容。
```

如果 AI 能返回知识库状态和文献列表，说明接入成功。

## 写论文与综述

可以这样开始：

```text
请基于我的知识库，写一篇关于未成年人分流转处的文献综述。
先检索和深读核心文献，再给我三级提纲。
正式起草前先让我确认提纲。
所有实质论断都要能回溯到原文位置。
```

典型流程是：

```text
拆解选题
→ 检索相关文献
→ 制作文献地图和提纲
→ 深读关键来源
→ 逐节起草
→ 核对论断与引文
→ 形成交付物
→ 沉淀到综述库
```

## 维护知识库与综述库

可以这样说：

```text
请维护我的知识库和综述库。
先做全量检查，简单问题直接处理。
涉及花费 API 额度、删除内容或实质性结论取舍时再问我。
结束后复核并给出完整总结。
```

维护工作通常检查：

- 文献是否完整入库；
- 深索是否失败或遗漏；
- 检索摘要是否缺失或异常；
- 新文献是否影响已有综述；
- 综述是否有断链、孤儿页或过时页；
- 项目记忆是否仍然准确。

## 跨学科发散与补文献

可以这样说：

```text
帮我把“罪错未成年人分流”这个题目做一次跨学科发散。
先看我的库已经覆盖了哪些理论，再指出应补哪些学科和外文文献。
这一步先做研究地图，不要直接写成论文。
```

Agent 会尝试把窄题连接到发展心理学、犯罪学、教育学、社会工作、公共管理或行为科学等邻近领域，并区分“库内已有证据”和“建议外部补充的方向”。

## 成果保存在哪里

### 交付物

论文、报告、资料汇编和周报等成品保存在：

```text
0_Agent交付物
```

每个研究主题应使用独立子文件夹。

### 综述库

可以复用的结论会保存为综述页，参与以后检索和维护。

### 项目记忆

研究偏好、已拍板决策和当前进度保存在本地的人类可读文件中。以后更换 AI 助手，新助手仍可读取同一套记忆和工作流。

## 定时任务

你可以让 AI 助手建立周报或定期跟踪任务。任务定义会保存在 PaperPiggy 的资料库中，成果进入交付物目录。

需要注意：PaperPiggy 负责登记、展示和接入知识库，真正的定时触发由 AI 助手自己的日程功能执行。

## 安全边界

正常研究工作默认是“只读原文，另存综合”：

- 不修改 Zotero；
- 不修改原始文献；
- 不让 Agent 覆盖人工核验页面；
- 涉及删除、付费调用或实质性结论取舍时，应先由用户决定。

接入 Agent 后，还应同时了解该 AI 助手自身的数据政策和权限范围。
