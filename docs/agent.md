## Agent 到底是什么

Agent 是能够自己调用工具、连续完成多步任务的 AI 助手。

你对普通聊天说“帮我写一段综述”，它往往只能根据当前对话和通用知识直接回答。你对接入 PaperPiggy 的 Agent 说同一句话，它可以：

1. 先检查知识库状态；
2. 检索相关文献；
3. 选择最值得精读的来源；
4. 读取带页码或章节位置的原文；
5. 核对引文和实质论断；
6. 生成综述、提纲或资料汇编；
7. 把成果保存到 Agent 交付物或综述库；
8. 记录资料缺口和下一步任务。

## 可以连接哪些 Agent

只要客户端支持 MCP，就可以连接 PaperPiggy。应用当前重点提供：

- **Claude Code**：应用内推荐的入门选项；
- **Codex**：同样可以接入；
- **其他支持 MCP 的客户端**：使用 Agent 页提供的通用配置。

Agent 使用的模型和账户由对应客户端决定。PaperPiggy 里的硅基流动 Key 主要服务于知识库检索和应用内辅助功能，两者不要混为一谈。

## 连接前检查

<ul class="checklist">
  <li>PaperPiggy 已经完成 Zotero 连接和第一版索引。</li>
  <li>Claude Code、Codex 或其他 MCP 客户端已经可以正常打开。</li>
  <li>你知道自己准备在哪个 Agent 客户端中使用知识库。</li>
</ul>

## 从应用内复制真实接入命令

<div class="steps" markdown="1">

### 打开 Agent 页

在 PaperPiggy 顶栏点击“Agent”。

### 选择客户端

选择 Claude Code、Codex 或通用 MCP 配置。应用会显示与你当前电脑和安装目录相匹配的接入命令。

### 复制并运行

在任意对应的 Agent 会话中粘贴运行。接入命令只需要执行一次；之后新会话会自动发现 PaperPiggy。

### 重新进入会话

如果当前会话没有立即看到新工具，退出后重新打开一个会话，或按客户端方式刷新 MCP 连接。

</div>

<div class="callout">
  不要从网页教程手抄路径、端口或配置文件内容。PaperPiggy 的 Agent 页会生成本机真实命令，那里才是接入配置的唯一可靠来源。
</div>

## 第一次验证

接入后，先做一个不写入内容的简单检查：

<div class="prompt">连接我的 PaperPiggy 知识库，先查看知识库状态，再搜索“一个确定存在于我文库中的主题”。告诉我你实际调用了哪些知识库能力，不要只用通用知识回答。</div>

正常情况下，Agent 会返回文献数量、索引或深索状态，并列出库内真实文献。

## 不需要另外安装工作流

PaperPiggy 已经准备了三条开箱即用的工作流：

- 写论文与综述；
- 维护综述库；
- 跨学科发散与补文献。

当你的请求与某条工作流匹配时，Agent 会先读取工作流再开始执行。工作流都是本机上的 Markdown 文件，以后可以直接与 Agent 商量修改，逐渐变成你自己的研究方法。

## Agent 能改什么，不能改什么

Agent 可以：

- 检索和读取知识库；
- 发起增量建库、深索和检索摘要维护；
- 创建或更新由 Agent 维护的综述页；
- 建立知识页之间的链接；
- 标记可能过时的结论；
- 写入交付物、资料库和项目记忆。

Agent 不会：

- 修改 Zotero 中的原始条目和附件；
- 删除原始文献；
- 通过 MCP 删除综述页；
- 覆盖你已经人工核验的知识页。

综述库还保留版本历史。Agent 写错时，可以在应用中查看历史并回滚。

## 建议的第一次正式任务

<div class="prompt">围绕“我的当前课题”，先检查知识库覆盖情况，搜索相关文献并分组。只精读最相关的 5 篇，逐篇给出原文位置和核心论点；如果文献还未深索，先完成这 5 篇的深索和检索摘要。最后列出已有证据、相互冲突的观点和仍需补充的资料。</div>

这个任务能同时验证检索、深读、原文定位和检索摘要，也不会一开始就让整个大库长时间运行。
