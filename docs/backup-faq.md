## 先区分三件事

| 操作 | 作用 | 什么时候用 |
|---|---|---|
| 立即更新知识库 | 发现新增或变化的文献并补齐处理 | 日常同步、版本升级后发现新格式 |
| 清空并从头重建索引 | 重新生成索引产物 | 索引严重异常，且普通更新无法修复 |
| 应用更新 | 把 PaperPiggy 程序升级到新版 | 有新版本时 |

它们互不等价。新增几篇文献不需要升级应用；升级到支持五种全文格式的版本后，也不需要清空旧 PDF 索引。

## 备份

在设置页打开“备份与恢复”，先创建备份，再把备份文件复制到另一个可靠位置。

建议在这些时点备份：

- 第一次完整建库后；
- 清空并重建索引前；
- 大批量整理、迁移或替换文献前；
- 应用大版本更新前；
- 综述库或 Agent 交付物有重要进展后。

## 恢复前要做什么

1. 确认选中的是正确备份；
2. 记下当前数据是否还有未备份的新内容；
3. 关闭可能正在写入知识库的外部 AI 助手；
4. 按应用提示执行恢复并重新打开；
5. 随机抽查文献、综述库和设置。

恢复会改变现有数据状态，不要把“试试看”当成无风险操作。

## 应用更新

设置页的“应用更新”可以检查、下载并应用新版本。更新前建议：

- 先看版本说明；
- 重要数据先备份；
- 下载和替换期间不要强制关机；
- 如果你修改过应用源码，先保留自己的修改。

应用内更新只处理程序区域，设计上不会把知识库数据当作更新目标；完整安装器覆盖安装时，仍应按提示保护自己的自定义代码。

## 常见问题

### 新增文献为什么搜不到？

先执行“立即更新知识库”，再观察后台任务。题录可能先出现，语义索引和全文处理稍后完成。

### EPUB、DOCX、Markdown 或 TXT 为什么没有出现？

确认应用已更新到支持五种全文格式的版本，然后执行一次“立即更新知识库”。不要先清空旧索引。HTML 当前不支持。

### PDF 能打开，深索却没有内容？

尝试复制一段正文。如果无法复制，文件可能是扫描件，需要 OCR；也可能是全文处理尚未完成。

### 提示 API 错误或余额不足怎么办？

核对 API Key、服务商账号、网络和余额。硅基流动账户即使某些模型标为免费，也可能在余额为零时拒绝请求。不要在截图或公开 issue 中粘贴完整 Key。

### 模型下载很慢怎么办？

GitHub 网络在部分地区较慢。保持应用开启并重试；避免同时启动多个实例或用清理工具删除临时下载。

### 点应用图标没有反应怎么办？

先查看任务管理器中是否已有 PaperPiggy 进程，再尝试重新打开。若仍异常，记录应用版本、操作步骤和日志信息后提交问题；不要直接修改正式安装目录里的文件。

### 什么时候才需要清空重建？

只有在应用明确提示、索引长期异常且“立即更新”无法修复，或版本说明明确要求时。执行前先备份。

## 提交问题时请提供

- PaperPiggy 版本；
- Windows 版本；
- 你刚才做了什么；
- 预期结果与实际结果；
- 可公开的报错文字或日志片段；
- 已经尝试过哪些步骤。

请删除 API Key、私人路径、未公开论文题目和个人信息。可以在 [主仓库 Issues](https://github.com/DrinkTea905/paper-piggy/issues) 报告应用问题，在 [教程仓库 Issues](https://github.com/DrinkTea905/paper-piggy-guide/issues) 反馈教程问题。
