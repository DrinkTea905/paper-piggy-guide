# 备份、更新与常见问题

## 一定要做备份

打开：

```text
⚙ 设置 → 数据与维护 → 备份与恢复
```

点击“立即备份”。备份会包含难以重新生成的重要资产，例如：

- 综述库；
- Agent 交付物和项目记忆；
- 手动分类；
- 文献元数据；
- 来源评价数据；
- 其他设置和人工成果。

### 是否包含向量索引

- 日常备份：通常可以不包含，备份包更小。
- 换电脑或不想重新建库：可以包含索引，体积会明显增大。

如果 API 模式重建大量向量会再次消耗额度，迁移前更应考虑包含索引。

### 是否包含 API Key

备份默认不包含 API Key。

除非非常清楚风险，否则不要勾选“包含 API 密钥”。包含密钥的备份包一旦上传、转发或丢失，就可能造成泄漏。

### 正确的云备份

可以把备份位置设到 OneDrive 或坚果云目录，让云盘同步生成后的静态 ZIP。

不要直接同步整个 PaperPiggy 安装目录或数据目录。

## 更新知识库与更新应用

### 更新知识库

顶栏的“⟳ 手动更新知识库”用于读取 Zotero 新增条目，或扫描受管文件夹中的新文件。

它会增量更新题录和语义索引，但不会自动完成所有全文深索。

也可以在“⚙ 设置 → 建库 → 自动更新知识库”设置按天检查。

### 更新应用

右上角出现“有新版”表示 PaperPiggy 程序本身有更新。

打开“⚙ 设置 → 应用 → 应用更新”即可检查。通常应用内更新只替换程序，不删除索引、综述、设置和 Agent 文件。

如果提示需要完整安装器，可以把新版安装到原来的位置，不必先卸载。

## 检索内存

PaperPiggy 平时只保留轻量目录，第一次检索时才加载模型和倒排索引。空闲一段时间后会自动释放，给其他软件腾内存。

可以在“⚙ 设置 → 检索 → 检索内存”调整：

- 内存紧张：5～10 分钟；
- 经常连续检索：30～60 分钟；
- 希望下次检索最快：不自动释放。

释放只影响临时内存，不删除文献、索引或设置。

## 常见问题

### 启动后没有文献

依次检查：

1. 是否已经连接 Zotero 或全文文件夹；
2. Zotero 数据目录是否正确；
3. 文件夹中是否有支持的全文格式；
4. 是否执行过“手动更新知识库”；
5. 是否开启“只导入有全文附件”，导致纯题录被跳过。

### 题录能找到，但全文搜索不到

通常是文献还没有完成深索。在文献页筛选“未深索”，选择目标文献并开始深索。

### 第一次检索等待较久

本地模式需要加载模型和索引。检索内存释放后，下一次检索也需要重新准备。

### API 测试返回 403

检查：

- API Key 是否完整；
- Base URL 是否正确；
- 账号是否有可用额度；
- 注册和登录使用的域名是否一致；
- 服务商当前是否允许调用所选模型。

免费额度和账户规则以服务商页面为准。

### 文件夹模式只有文件名

题录抽取可能没有正常工作。检查是否配置了可用的大模型 API Key，然后重新建库或补全题录。

### 扫描 PDF 没有正文

查看具体状态：等待 OCR、OCR 失败、附件丢失、文件损坏、原生文字层成功或 OCR 成功。不同状态需要分别处理。

### Agent 配好后没有看到知识库

配置通常在新会话中加载。关闭当前 AI 会话并重新打开，再确认是否能看到 `localkb`。

### 应用更新和知识库更新有什么区别

- 应用更新：更新 PaperPiggy 程序。
- 知识库更新：读取新增文献并更新索引。
- 深索：进一步读取全文，仍需按需进行。

### 是否应该“清空并从头重建索引”

这不是普通故障的第一解决方案。只有库内重复、深索状态严重混乱或确实需要整体重建时才考虑。

操作前先在“备份与恢复”中建立备份，最好包含向量索引。清空会丢弃向量并需要重新嵌入，API 模式可能再次产生费用。

## 导出错误日志

仍无法解决时，打开：

```text
⚙ 设置 → 应用 → 帮助与维护 → 查看 / 导出错误日志
```

复制错误信息，到 [PaperPiggy 主仓库问题区](https://github.com/DrinkTea905/paper-piggy/issues)反馈。提交前请检查日志中是否包含私人路径、文献名称或其他敏感信息。

## 推荐的长期使用节奏

1. 新增文献后更新知识库；
2. 优先深索当前课题的重要来源；
3. 定期处理摘要缺失和异常；
4. 把重要问答沉淀进综述库；
5. 让 Agent 定期维护综述和项目记忆；
6. 建立自动备份；
7. 大规模重建或迁移前手动备份一次。
