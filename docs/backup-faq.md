## 先分清三种“更新”

| 操作 | 实际作用 | 什么时候使用 |
|---|---|---|
| 手动更新知识库 | 扫描 Zotero 或文件夹的新增和变化，补题录与语义索引 | 日常新增文献后 |
| 清空并从头重建索引 | 删除并重新生成索引产物 | 索引严重异常且普通更新无效 |
| 应用更新 | 把 PaperPiggy 程序升级到新版本 | 设置页提示有新版时 |

新增文献后先更新知识库，不需要重新安装应用。普通版本升级也不需要顺手清空索引。

## 建议怎样备份

在“设置 → 数据与维护 → 备份与恢复”中创建备份。PaperPiggy 会把真正不可替代的内容打包，例如：

- 综述库；
- Agent 交付物、资料库和项目记忆；
- 收藏夹分类；
- 来源评价和人工修改；
- 其他用户生成的数据。

如需云备份，把生成的静态备份压缩包同步到 OneDrive、坚果云等位置即可。不要直接同步正在持续读写的整个数据目录。

备份过程中只要有应保存的文件无法读取，整次备份就会停止，不会留下一个“看似成功、实际缺文件”的压缩包。

建议在这些时点备份：

- 第一次完整建库后；
- 一个研究阶段结束后；
- 大批量整理文献前；
- 清空重建索引前；
- 应用大版本更新前。

## 怎样恢复

恢复会把数据回到备份时的状态。操作前先确认备份时间，并保存当前仍需保留的新成果。

PaperPiggy 会先检查压缩包是否完整、安全，拒绝损坏文件、越界路径和异常链接；随后在临时区域完整展开并验证。只有全部通过后才会替换现有数据。失败时不会改动当前数据；恢复前的数据会保留在可找回目录，替换失败也会自动回滚并保留现场。

恢复完成后，随机检查：

- 文献数量和分类；
- 综述库页面；
- Agent 项目记忆与交付物；
- 检索引擎和建库设置。

## 应用更新

设置页的“应用更新”可以检查、下载并应用新版。更新程序不会把知识库数据当作替换目标。

下载完成后，PaperPiggy 会核对官方 SHA-256、完整文件清单和每个文件的哈希；损坏、被篡改、缺文件或多出意外文件的更新包会被拒绝。新版先在临时区域验证，成功后才替换应用代码；失败时会回滚，用户自行修改过的旧代码和失败现场会保留。

如果应用内更新失败，可以从 [GitHub Releases](https://github.com/DrinkTea905/paper-piggy/releases/latest) 下载最新安装器，直接覆盖原安装位置，不必先卸载。

## 删除同步与清理

开启删除同步或手动清理已删除文献时，PaperPiggy 会先预览预计处理数量，再要求二次确认。它只清理 PaperPiggy 自己的索引记录，不会删除 Zotero 条目或原始文件。

如果一次预计清理超过 20 篇，或超过当前文库的一半，操作会自动中止。先检查 Zotero 或受管文件夹是否暂时无法读取，不要在来源异常时强行清理。

## 常见问题

### 硅基流动 Key 测试失败

依次检查：

1. 是否使用了正确账户创建的 Key；
2. <code>.cn</code> 与 <code>.com</code> 是否混用；
3. 账户是否完成实名认证；
4. 余额是否为零；
5. 网络能否访问硅基流动。

如果返回余额不足，即使模型标记为免费，也先充值 1 元再测试。

### Zotero 没有被自动识别

在 Zotero 的“首选项 → 高级 → 文件和文件夹”中查看数据目录，把包含 <code>zotero.sqlite</code> 的目录手动填入 PaperPiggy。

### Zotero 新文献没有出现

先点击“手动更新知识库”。题录会先出现，语义索引随后在后台完成；深索仍需要按优先级另行处理。

### 文献能看到，但 Agent 读不到正文

这通常表示只有题录，尚未深索。让 Agent 查看该文献状态，并完成深索和检索摘要。

<div class="prompt">检查《文献标题》的题录、全文附件、深索和检索摘要状态。能自动处理的直接完成；如果附件丢失、损坏或格式不支持，告诉我具体原因。</div>

### 扫描版 PDF 深索很慢

PaperPiggy 会自动在本机 OCR 没有文字层的页面。OCR 比普通文本提取慢得多，属于正常现象。可以先处理当前课题的重要文献，其余安排在电脑空闲时运行。

### 检索结果找不准

让 Agent 先检查：

- 语义索引是否完成；
- 关键文献是否已经深索；
- 检索摘要是否有效；
- 当前查询是否过宽或过窄；
- 是否需要限定 Zotero 分类、年份或来源类型。

不要第一反应就清空索引。

### Agent 看不到 PaperPiggy

回到应用的“Agent”页，重新复制对应客户端的接入命令。执行后重新打开 Agent 会话，再让它查看知识库状态。

### Agent 能连接，但只凭通用知识回答

在任务中明确要求它先检查 PaperPiggy 状态、检索库内文献并给出原文位置。也可以直接说：

<div class="prompt">这项任务必须使用我的 PaperPiggy 知识库。先调用知识库检查和检索能力，再回答；库内没有证据时请明确说没有，不要用通用知识补齐。</div>

### 深索全库要多久

时间取决于文献数量、文件大小、扫描件比例和电脑性能。几百到上千篇文献可能需要数小时甚至更久。先按课题分批处理，再利用夜间空闲时间补齐。

### 什么时候才需要清空重建

只有以下情况才考虑：

- 应用明确提示索引异常；
- 重复或错误索引无法通过普通更新修复；
- 更换了不兼容的检索引擎；
- 版本说明明确要求。

执行前先创建备份。

- Windows 版本；

- 发生问题前执行的步骤；
- 完整报错文字；
- 已经尝试过哪些处理。
