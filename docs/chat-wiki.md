# 内置对话与综述库

## 什么时候使用内置对话

“💬 对话”适合快速询问一个问题。PaperPiggy 会先检索本地知识库，再把相关来源交给你配置的大模型回答。

它适合：

- 随手理解一个概念；
- 比较几篇文献的观点；
- 从当前库中寻找支持和反对证据；
- 快速生成一个可以继续核对的答案。

需要跨会话积累、系统检索、长篇写作或持续维护时，更适合使用 Agent。

## 配置对话模型

第一次使用时，展开“⚙ 模型设置”，填写：

- 服务商；
- Base URL；
- API Key；
- 模型名称。

如果检索引擎已经配置 SiliconFlow Key，对话页通常可以复用。

问题和检索到的上下文会发送给你选择的大模型服务商。处理敏感文献前，请先确认是否适合使用外部模型。

## 第一次提问

尽量写清任务和证据要求。例如：

```text
请根据我的文献库，总结恢复性司法对未成年人再犯的影响。
区分支持证据与质疑证据，并给出可回溯的原文位置。
如果库内证据不足，请明确说明。
```

一个可靠回答至少应该：

- 说明使用了哪些文献；
- 给出页码、章节、段落或行号；
- 区分原文结论和模型综合；
- 没有证据时承认证据不足；
- 不把“我的库里没有”写成“学界不存在”。

重要论断和引文仍应打开原文核对。不要只因为回答带了一个页码，就默认它一定准确。

## 保存重要答案

如果答案值得长期保留，可以点击“保存此答案”，将其沉淀进“📖 综述库”。

内置对话的答案默认不会自动保存。只有明确保存后，它才会成为可以继续维护的知识页。

## 综述库是什么

综述库不是普通聊天记录，而是一组可以长期复用的研究页面。页面可以包含：

- 综合正文；
- 所用来源；
- 原文位置；
- 页面之间的链接；
- 核验状态；
- 更新时间和历史版本。

## 主题、筛选和关系图

综述页会根据来源和已有研究主题自动归类。你也可以：

- 新建自定义主题；
- 手动固定页面；
- 搜索标题；
- 按页面类型筛选；
- 只看尚未核验的页面；
- 按更新时间或来源数量排序；
- 查看页面关系图和时间线。

删除自定义主题只会恢复自动归类，不会顺便删除其中的综述页。

## 核验和过时状态

Agent 创建的页面会标记为“未核验”，并在检索中降低权重，不会轻易盖过原始文献。

人工核验过的页面受到保护，Agent 不能直接覆盖。

新文献入库后，如果可能影响旧结论，PaperPiggy 会生成更新建议。是否修改以及如何修改，仍需要根据新证据判断。

## 体检综述库

点击“体检综述库”，可以检查：

- 孤儿页；
- 过时页；
- 断链；
- 来源错误；
- 重复页面；
- 缺失的重要概念。

综述库的目标不是代替原文，而是保存“已经读过、综合过和核对过什么”，减少以后重复劳动。
