## 知识库需要持续维护

第一次建库只是开始。随着 Zotero 新增文献，旧综述可能过时，新的证据也可能影响多个既有结论。

维护不等于每次都手工检查所有页面。连接 Agent 后，只要明确提出“维护知识库”，它会先运行完整审查，再处理能够自动完成的事项。

<div class="prompt">维护我的 PaperPiggy 知识库。先做完整审查：检查新增题录、索引、全文附件、深索、OCR、检索摘要、待更新综述页、孤儿页、断链和过时内容。能安全自动处理的直接处理；需要我做真实内容取舍或涉及外部条件的事项，再集中向我说明。</div>

## Agent 会检查什么

### 文献是否真正可用

- Zotero 新增文献是否已经入库；
- 题录和附件是否齐全；
- 哪些重要文献还没有深索；
- 扫描件是否已经完成 OCR；
- 哪些文件损坏、丢失或无法读取。

### 检索摘要是否完整

检索摘要是完整知识库的一部分。Agent 会检查摘要是否缺失、异常或已经失效，并为可处理的文献读取正文、生成合格摘要、重新嵌入。

在“设置 → 建库 → 检索摘要”选择“交给 Agent 生成”后，Agent 会读取正文、生成摘要并完成重新嵌入。

### 综述库是否健康

Agent 会检查：

- 新文献可能影响哪些旧综述；
- 哪些页面已经过时；
- 哪些知识页没有来源；
- 哪些页面彼此孤立；
- 是否存在断链、重复主题或尚未建立独立页面的重要概念。

## 综述库在整个流程中的位置

综述库不是普通聊天记录。它用来保存需要反复使用和继续更新的研究成果，例如：

- 一个课题的总体论证地图；
- 一个制度的主要争议；
- 一组判例的裁判路线；
- 不同学科对同一概念的解释；
- 资料覆盖程度和下一步补文献计划。

网页端的“综述库”用于阅读、直接编辑正文、查看历史、切换核验状态和回滚。打开一页后点“编辑正文”即可随手修正；保存后自动标为已核验，也可以再改回未核验继续复查。标题、参考来源和版本历史由 PaperPiggy 自动维护。综述的创建、系统性更新、建链接和标记过时，主要交给 Agent 完成。

## 一篇新文献可能影响很多旧结论

Agent 读完一篇新文献后，不应只做一张孤立摘要。更完整的处理是：

1. 找出它直接支持或反驳的既有综述；
2. 找出讨论同一主题但尚未引用它的页面；
3. 更新相关页面或标记过时；
4. 建立知识页之间的链接；
5. 必要时更新总论页；
6. 记录还有哪些问题没有覆盖。

<div class="prompt">精读这篇新文献后，不要只做摘要。检查它影响了哪些既有综述页，逐页提出或完成更新，补齐页面之间的链接，并说明哪些旧结论被支持、修正或推翻。</div>

## 项目记忆与工作区

PaperPiggy 会为 Agent 准备本地工作区：

- **项目记忆**：当前有效的偏好、决策和研究进度；
- **变更日志**：历史记录；
- **技能**：可以修改的研究工作流；
- **资料库**：中间材料和可复用资料；
- **交付物**：论文初稿、资料汇编、报告等成果。

这些文件都是人类可读的。换一个 Agent 客户端时，新助手可以先读取项目记忆和工作流，继续之前的研究，而不是从零开始。

## 建议的维护节奏

### 每次加入一批新文献后

<div class="prompt">更新知识库，检查这批 Zotero 新文献是否已入库；按当前课题优先级完成必要深索和检索摘要，并处理它们带来的综述更新建议。</div>

### 每周或每个研究阶段结束时

<div class="prompt">做一次完整知识库维护。重点检查深索和检索摘要缺口、过时综述、孤儿页、断链、项目记忆和当前交付物；最后给出本次已处理事项和下一阶段建议。</div>

### 正式写作前

<div class="prompt">按照“写论文与综述”工作流体检当前课题。检查关键结论的来源覆盖、冲突证据和引文位置；把证据薄弱的章节标出来，不要把库内资料不足写成学界空白。</div>

## 需要你决定的事情

Agent 可以处理机械维护，但研究判断仍由你决定，例如：

- 论文究竟采取哪条论证主线；
- 冲突观点中采用哪种立场；
- 是否补充库外文献；
- 哪些内容达到人工核验标准；
- 最终交付物如何取舍和发布。

最理想的分工不是让 AI 替你做研究判断，而是让它承担重复的查找、对照、核验和整理工作，把你的时间留给真正的学术选择。
