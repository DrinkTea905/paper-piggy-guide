# 安装与首次设置

## 系统要求

普通用户目前使用 Windows 10/11 64 位安装器。安装包自带运行环境，不需要提前安装 Python。

macOS 和 Linux 暂时没有普通用户安装器，技术型用户可以参考程序主仓库的源码运行说明。

## 下载

打开 [PaperPiggy 最新版](https://github.com/DrinkTea905/paper-piggy/releases/latest)，下载 `PaperPiggy-x.y.z-win64.exe`。

版本号会变化，因此教程不写死某个版本。始终以“Latest”页面提供的安装器为准。

## 安装位置与数据位置

PaperPiggy 通常把程序、索引、模型、综述库和 Agent 工作区放在同一个安装目录中。

如果选择：

```text
D:\PaperPiggy
```

那么主要数据也会保存在这个目录附近，便于整体迁移和备份。

### 不要安装在云盘同步目录

不要把 PaperPiggy 安装在 OneDrive、坚果云、Dropbox 等目录，也不要事后把整个数据目录加入实时同步。

想做云备份时，应在 PaperPiggy 中生成静态 ZIP，再让云盘同步这个 ZIP。

## 首次向导

首次启动包含几个连续任务：

1. 检查 Zotero 环境；
2. 配置可选的 SiliconFlow Key；
3. 选择本地或 API 检索引擎；
4. 连接 Zotero 或全文文件夹；
5. 建立第一个索引。

### API Key 是否必须

- Zotero 题录入库：不一定需要。
- 文件夹模式自动识别题录：强烈建议配置。
- API 检索模式：必须配置并测试通过。
- 内置对话：需要大模型服务商的 Key。
- 本地检索模式：嵌入和重排不需要外部 Key。
- 服务端自动生成检索摘要：需要 Key，并可能消耗额度。

服务商的免费模型、额度和计费规则会变化，实际以服务商页面为准。

## 本地模式与 API 模式

| | 本地模式 | API 模式 |
|---|---|---|
| 首次准备 | 下载约 1GB 模型 | 填写并测试 API Key |
| 网络 | 模型就绪后可离线检索 | 建库与检索时需要联网 |
| 磁盘/内存 | 占用较多 | 占用较少 |
| 文本去向 | 嵌入和重排在本机 | 必要文本发送给所选服务商 |
| 费用 | 不按调用次数计费 | 取决于服务商规则 |

## 重看首次向导

以后想重新查看，可以打开：

```text
⚙ 设置 → 应用 → 帮助与维护 → 重看安装引导
```

需要注意：重看向导不代表可以无代价切换检索引擎。切换本地/API模式会使旧向量不兼容，应用会要求重建。
