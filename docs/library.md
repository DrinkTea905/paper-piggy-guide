# 连接文献与建立索引

PaperPiggy 支持两种文献来源：Zotero 和普通全文文件夹。二者选择一种即可。

## 路线 A：连接 Zotero

如果你已经使用 Zotero 管理文献，建议优先选择这一方式。

PaperPiggy 会读取 Zotero 的题录、全文附件和收藏夹结构，但不会修改 Zotero 数据。

### 检查数据目录

如果向导已经自动探测到 Zotero，可以直接连接。

没有自动探测到时，请填写包含下面文件的数据目录：

```text
zotero.sqlite
```

可以在 Zotero 的“首选项 → 高级 → 文件和文件夹”中查看数据目录。

### 是否只导入有全文附件的文献

勾选“只导入有全文附件的文献”后，只有带 PDF、EPUB、DOCX、Markdown 或 TXT 的条目会进入知识库，没有这些附件的纯题录会被跳过。

- 希望库中每篇都能进一步深索：建议勾选。
- 也想保留暂时没有全文的题录：可以不勾选。

多附件情况下，PaperPiggy 会按照 PDF、EPUB、DOCX、Markdown、TXT 的顺序选择主要全文。

连接成功后，界面会显示 Zotero 总条目数和准备实际导入的数量。

## 路线 B：建立文件夹知识库

没有使用 Zotero 时，可以指定一个普通文件夹，例如：

```text
D:\我的论文库
```

点击“打开全文文件夹”，放入支持的文件，然后返回 PaperPiggy，点击“建立文件夹库”。

### 为什么文件夹模式最好配置 API Key

普通文件通常没有完整的结构化题录。PaperPiggy 需要从正文开头识别题名、作者、年份、期刊和摘要。

配置 API Key 后，系统会自动抽取这些信息。没有 Key 仍可入库，但可能只能显示文件名，查找、分类和来源评价效果会下降。

以后也可以直接把全文文件拖入“📚 文献”页面，系统会复制到受管文件夹并入库。

## 建立第一个索引

### Zotero 模式

PaperPiggy 会先建立轻量题录索引。完成后，按标题、作者、年份等查找会立即可用。

随后会在后台继续建立语义索引。你可以先进入知识库，不必一直停在向导中等待。

### 文件夹模式

系统需要逐篇识别题录，因此比 Zotero 模式更慢。顶部进度条会显示后台建库进度。

<div class="check"><b>成功标志：</b>“📊 库总览”有文献数量，“📚 文献”页能显示条目，按作者或题名能返回结果。</div>

## 认识文献页面

文献页面把浏览、筛选和检索放在同一页。

左侧包含：

- 全部文献；
- 手动分类；
- AI 分类；
- Zotero 分类（仅 Zotero 模式）。

右侧可以按以下条件筛选：

- 深索状态；
- 正文来源；
- 检索摘要状态；
- 文献性质；
- 客观来源标签；
- 入库时间、年份或综合推荐。

## 题录查找

保持搜索模式为“题录查找”，输入：

- 题名；
- 作者；
- 期刊；
- 年份；
- DOI；
- ISBN。

题录查找不要求文献完成深索，适合确认“库里有没有这篇”。

## 更新文献来源

在 Zotero 新增文献或向受管文件夹放入新文件后，点击顶栏：

```text
⟳ 手动更新知识库
```

这会增量读取新增内容，并更新题录和语义索引；已经处理过的文献会跳过。深索仍需手动选择。

也可以在“⚙ 设置 → 建库 → 自动更新知识库”中设置按天检查。
