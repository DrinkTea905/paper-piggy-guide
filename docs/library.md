## 支持哪些全文格式

当前支持五种全文格式：

| 格式 | 常见用途 | 深索定位方式 |
|---|---|---|
| PDF | 论文、报告、法源 | 页码 |
| EPUB | 电子书 | 章节 |
| DOCX | 草稿、报告、内部材料 | 标题与段落 |
| Markdown | 笔记、知识文档 | 标题与段落 |
| TXT | 纯文本资料 | 行号范围 |

HTML 不属于当前支持格式。同一条记录有多个附件时，主全文通常按 PDF → EPUB → DOCX → Markdown → TXT 的顺序选择。

<div class="callout">
  <strong>从旧版升级的用户：</strong>已有 PDF 索引不需要清空重建。升级后执行一次“立即更新知识库”，让应用发现新增的 EPUB、DOCX、Markdown 和 TXT 文件即可。
</div>

## 一篇文献是怎样进入知识库的

<div class="flow">
  <span>扫描题录</span>
  <span>建立语义索引</span>
  <span>处理全文</span>
  <span>生成检索摘要</span>
</div>

这四层不是同一件事：

1. **题录层**让标题、作者、年份尽快可见。
2. **语义层**让你能按概念找文献，而不只依赖原词。
3. **全文层**为深索准备可定位的正文片段。
4. **检索摘要层**是可选增强，用于改善检索理解，可能调用 API。

题录先出现、全文稍后完成是正常现象。

## 使用 Zotero 建库

<div class="steps">

### 先整理 Zotero

确认重要条目至少有标题、作者、年份，并且附件能够正常打开。重复条目、空附件和损坏文件会降低后续体验。

### 在 PaperPiggy 选择 Zotero

按界面提示定位 Zotero 数据。若自动识别失败，再手动选择，不要凭印象改系统文件。

### 第一次只导入一部分

如果文库很大，先验证一个集合或少量文献。确认题录、语义检索和深索都正常，再扩大范围。

</div>

## 使用普通文件夹建库

- 选择真正存放文献的上层文件夹；
- 可以保留子文件夹，PaperPiggy 会按实际扫描规则处理；
- 不要在扫描过程中大量改名或移动文件；
- 建库完成后，用一篇已知文献做标题检索和深索验证。

## “立即更新”与“清空重建”

- **立即更新知识库**：发现新增或变化的文件，适合日常同步和版本升级后补扫。
- **清空并从头重建索引**：用于索引严重异常或明确需要重新生成的情况，不是普通刷新按钮。

<div class="callout warn">
  不确定时先用“立即更新”。清空重建会花更长时间；虽然设计上保留综述库、分类、检索摘要等核心资料，也仍应先做备份。
</div>

## 扫描件和 OCR

有些 PDF 看起来有文字，实际只有图片。判断方法：尝试在阅读器里选择并复制一段文字。如果复制不到正文，深索可能也读不到，需要先做 OCR。

## 建库完成检查

<ul class="checklist">
  <li>随机抽查几篇文献的标题、作者和年份。</li>
  <li>用一个主题词做普通检索，检查结果是否合理。</li>
  <li>挑一篇全文做深索，检查定位是否能回到原文。</li>
  <li>记录未识别、损坏或需要 OCR 的文件，逐步处理。</li>
</ul>
