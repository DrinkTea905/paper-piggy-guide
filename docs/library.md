## 为什么推荐 Zotero

Zotero 是 PaperPiggy 推荐的文献来源。使用 Zotero 时，它可以直接读取：

- 题名、作者、年份、期刊等结构化题录；
- 收藏夹和分类；
- 与条目关联的全文附件；
- 后续在 Zotero 中新增或更新的文献。

PaperPiggy 只读取这些数据，不会修改 Zotero 文库。没有使用 Zotero 时才考虑文件夹模式；文件夹模式的限制见[进阶设置]({{ '/advanced/' | relative_url }})。

## 建库前整理 Zotero

不需要先把整个文库整理得完美，但建议做到：

- 当前课题的重要文献已经进入 Zotero；
- 重要条目至少有题名、作者和年份；
- 需要精读的文献带有可正常打开的全文附件；
- 重复条目和损坏附件尽量先处理。

支持的全文格式包括 PDF、EPUB、DOCX、Markdown 和 TXT。HTML 暂不支持。

## 连接 Zotero

在首次向导的“连接文库”步骤选择“连接 Zotero”。

PaperPiggy 通常会自动找到 Zotero 数据目录。如果没有自动找到，在 Zotero 的“首选项 → 高级 → 文件和文件夹”中查看数据目录，然后把包含 <code>zotero.sqlite</code> 的目录填入 PaperPiggy。

点击“连接文库”后，先检查识别到的文献数量是否大致合理，再进入下一步。

## 是否只导入带全文附件的条目

如果你希望 Agent 能直接读原文、核对引文，建议勾选“只导入有全文附件的文献”。这样纯题录不会进入知识库。

如果你还希望检索那些暂时没有附件的书目，可以取消勾选；但这些条目只能提供题录和摘要，不能深索原文。

## 知识库的建设过程

<div class="flow">
  <span>题录索引</span>
  <span>语义索引</span>
  <span>全文深索</span>
  <span>检索摘要</span>
</div>

### 1. 题录索引：先让文献可见

PaperPiggy 先为题名、作者、摘要和关键词建立轻量索引。Zotero 模式通常很快完成，完成后即可浏览文献。

### 2. 语义索引：让概念能够被找到

语义索引在后台继续建立。它能找到表达不同、含义相关的文献，例如查询“平台控制劳动过程”，也可能找到使用“算法管理”表述的文献。

### 3. 全文深索：让 Agent 真正读原文

深索会提取全文、切成可检索片段，并保留原文位置：

- PDF 定位到页码；
- EPUB 定位到章节；
- DOCX、Markdown 定位到标题和段落；
- TXT 定位到行号范围。

扫描版 PDF 会在深索时自动进行本地 OCR，不需要提前另做一次导入。OCR 和全文处理都比较耗时，因此深索本来就是慢任务。

### 4. 检索摘要：可选的召回增强

深索完成后，可以给文献配一段约 150 字的检索摘要。它不是原文摘要，而是专门用于帮助语义检索识别这篇文献的主题、方法和主要结论。

检索摘要不能替代深索，也不是读取全文的前提：没有摘要，Agent 仍然可以精读、定位和引用已经深索的全文，只是整篇文献的召回可能不够稳定。推荐在“设置 → 建库 → 检索摘要”选择“交给 Agent 生成”；如果希望在应用内点深索后自动生成，可选择“PaperPiggy 自动生成”，再决定复用 SiliconFlow 免费简单模型，或使用其他通常收费的 AI 厂商和自选模型。

## 不要第一次就深索全库

如果 Zotero 里有几百或上千篇文献，整库深索可能持续数小时甚至更久。第一次进入主界面后：

1. 先完成 Agent 连接；
2. 让 Agent 检查当前研究方向需要哪些文献；
3. 优先深索当前课题、常用分类和最近新增文献；
4. 电脑夜间空闲时，再逐批处理剩余文库。

进入“文献”页后，左侧可以看到从 Zotero 同步过来的分类。先点击与你当前课题对应的分类，页面会只显示该分类中的文献；再从中勾选需要优先精读的文献，点击“深索选中”。分类名称后的两个数字表示“已深索篇数 / 该分类总篇数”，可以直接看出这一组文献还剩多少没有处理。

<div class="prompt">检查我的 PaperPiggy 知识库。根据当前课题，选出最值得优先深索的文献，分批完成深索和检索摘要。先处理一小批并报告结果，不要直接启动整库任务。</div>

## 日后新增文献

继续像平时一样在 Zotero 中保存文献和附件。随后可以：

- 点击 PaperPiggy 顶栏的“手动更新知识库”；
- 在设置中启用“自动更新知识库”；
- 直接让 Agent 执行一次知识库维护。

新增题录会先入库，深索仍按你的研究优先级逐步完成。

## 建库完成检查

<ul class="checklist">
  <li>识别到的文献数量与 Zotero 大致一致。</li>
  <li>“文献”页可以看到标题、作者和分类。</li>
  <li>语义索引已经开始或完成。</li>
  <li>已经决定先深索哪些研究方向，而不是立即跑完整个大库。</li>
</ul>
