# 10 分钟快速开始

这一页只带你完成一件事：**让第一批文献进入 PaperPiggy，并成功搜到它们。**

## 第 1 步：下载安装器

打开 [PaperPiggy 最新版下载页](https://github.com/DrinkTea905/paper-piggy/releases/latest)，下载名称类似下面的文件：

```text
PaperPiggy-x.y.z-win64.exe
```

不要下载“Source code”压缩包。

安装时可以使用默认位置，也可以选择空间充足的本地磁盘，例如 `D:\PaperPiggy`。

<div class="warn">不要安装到 OneDrive、坚果云、Dropbox 等云盘同步目录。索引是持续读写的数据库，实时同步可能造成文件冲突和损坏。</div>

<div class="shot">截图待补：最新版下载页面和安装位置选择页面</div>

## 第 2 步：选择检索引擎

首次启动时，PaperPiggy 会打开分步向导。

### 想最快开始：选择 API 模式

填写 SiliconFlow 等兼容服务的 API Key，点击“测试连接”。测试通过后再继续。

优点是不用下载本地模型，占用磁盘和内存较少；缺点是建库和检索时需要联网，必要文本会发送给服务商。

### 想离线检索：选择本地模式

本地模式首次需要下载约 1GB 模型，具体大小以当前界面提示为准。下载完成后，嵌入和重排可以在本机离线运行。

<div class="shot">截图待补：本地模式与 API 模式选择页面</div>

<div class="warn">已经建库后再切换本地/API模式，旧向量将不兼容，需要重新建立索引。第一次选择时不必追求“最强”，先选最符合隐私和设备条件的一种。</div>

## 第 3 步：选择文献来源

### 已经使用 Zotero

选择“连接 Zotero”。如果系统没有自动探测到 Zotero，请填写包含 `zotero.sqlite` 的数据目录。

PaperPiggy 只读取 Zotero，不会修改其中的题录、附件或收藏夹。

### 没有使用 Zotero

选择“文件夹模式”，指定一个存放 PDF、EPUB、DOCX、Markdown 或 TXT 的文件夹。

文件夹模式最好配置 API Key，否则自动识别题名、作者和年份的效果会受影响；没有 Key 时仍可入库，但可能只显示文件名。

<div class="shot">截图待补：Zotero与文件夹模式选择页面</div>

## 第 4 步：建立第一个索引

连接来源后，点击向导中的“建立索引”或“开始建库”。

- Zotero 模式会先快速建立题录索引，随后在后台提升语义检索质量。
- 文件夹模式还要逐篇识别题录，因此通常更慢，可以让它在后台继续。

成功后点击“进入知识库”。

## 第 5 步：做一次成功检查

打开“📚 文献”，确认：

1. 页面上已经出现文献；
2. 搜索模式为“题录查找”；
3. 输入一个作者或标题能够返回结果；
4. Zotero 用户能在左侧看到收藏夹。

<div class="check"><b>完成检查：</b>文献页已经出现条目，而且按作者或题名能找到文献。满足这两个条件，就已经完成第一次建库。</div>

## 接下来做什么

建议不要立刻把全库全部深索。先从当前课题最重要的几十篇开始：

1. 在文献页筛选“未深索”；
2. 勾选重要文献；
3. 点击“深索选中”；
4. 完成后切换到“全文与相关内容”；
5. 输入一个完整研究问题。

详细操作见[深索与全文检索]({{ '/search/' | relative_url }})。
