<div class="callout tip">
  <strong>本章目标：</strong>看到至少一条检索结果，并完成一次带来源的知识库问答。第一次只导入一个小文件夹，会更容易判断问题出在哪里。
</div>

## 开始前准备

- 一台 Windows 电脑。
- 5～20 篇用于测试的文献；支持 PDF、EPUB、DOCX、Markdown、TXT。
- 至少一份可以正常打开并包含正文的文件。
- 如果要使用在线 AI 能力，再准备硅基流动 API Key；只做本地建库和本地检索可以暂不填写。

## 十分钟流程

<div class="steps">

### 下载并安装

前往 [PaperPiggy 最新版本](https://github.com/DrinkTea905/paper-piggy/releases/latest)，下载安装器。建议装到空间充足、自己有写入权限的位置。

<div class="shot">安装器中的安装位置选择</div>

### 完成首次启动

第一次启动会引导你检查模型。模型下载量约 1 GB，实际大小以应用界面为准。下载期间不要反复关闭应用。

<div class="shot">首次启动向导与模型下载状态</div>

### 建一个小型测试库

选择“普通文件夹”，指定刚才准备的测试文件夹。先让 PaperPiggy 扫描题录；语义索引和全文处理可以继续在后台完成。

<div class="callout">
  如果你平时用 Zotero，也可以直接选择 Zotero。快速体验阶段用小文件夹更容易理解每一步。
</div>

### 先做一次普通检索

打开“文献”页，输入一个明确主题，例如“算法歧视 举证责任”。先观察标题、作者、年份和来源，再打开最相关的文献。

### 再做一次深索

对最相关的文献启动深索，输入更具体的问题，例如“作者如何论证举证责任倒置”。深索适合定位正文段落，不等于全文搜索框。

### 做一次知识库问答

打开问答页，提出一个范围清楚的问题，并要求回答附来源。例如：

> 根据我的知识库，比较两篇文献对平台责任边界的不同观点，并列出来源。

核对引用是否真的支持对应结论。没有合适来源时，不要把回答当成事实。

</div>

## 你应该看到什么

<ul class="checklist">
  <li>“文献”页能看到已导入的题录。</li>
  <li>普通检索能返回与你的关键词或概念相关的结果。</li>
  <li>深索能定位到正文中的具体片段。</li>
  <li>问答能给出来源，且来源可以回到原文核对。</li>
</ul>

## 如果没跑通

| 现象 | 先检查 |
|---|---|
| 一篇文献都没有 | 选中的文件夹是否正确；文件格式是否受支持 |
| 能看到标题但搜不到正文 | 语义索引或全文处理是否还在进行 |
| 扫描件没有文字 | 是否需要 OCR；原文件能否复制出正文 |
| 在线功能报错 | API Key、账户余额和网络是否正常 |
| 应用提示需要更新知识库 | 先点“立即更新”，不要直接清空索引 |
