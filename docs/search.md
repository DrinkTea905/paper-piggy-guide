## 两种检索解决不同问题

| 工具 | 最适合回答 | 结果粒度 |
|---|---|---|
| 文献检索 | 哪些文献与主题相关？ | 文献级 |
| 深索 | 某篇或一组文献中，哪段正文回答了问题？ | 段落级 |

一个可靠的研究流程通常是：先用文献检索缩小范围，再用深索核对证据。

## 写好一个文献检索式

优先写“研究对象 + 核心问题 + 必要限定”，例如：

- 平台用工 劳动关系认定 算法管理
- 生成式人工智能 著作权 合理使用
- 个人信息保护 同意规则 比较法

不必把问题写成很长的一句话。第一次结果太窄时，删除次要限定；太宽时，再加法域、年份或制度名称。

## 怎么看结果

不要只盯着第一名。建议依次检查：

1. 标题和摘要是否真正相关；
2. 来源类型、年份和期刊信息是否适合你的问题；
3. 结果命中了关键词，还是语义上回答了问题；
4. 打开原文后，论证是否与检索摘要一致。

PaperPiggy 会综合语义、关键词、重排和来源评价等信息，但排序不是学术判断的替代品。

## 深索的正确用法

深索问题要具体、可在正文中找到依据。例如：

- 作者对“实质性替代”提出了哪些判断标准？
- 本文如何区分平台责任与用户责任？
- 哪几段讨论了举证责任的转移？
- 结论依赖哪些案例或实证材料？

<div class="callout tip">
  先问一个证据问题，再要求比较或总结。一次塞入五六个子问题，往往会让命中片段变得松散。
</div>

## 不同格式的定位

- PDF：页码；
- EPUB：章节；
- DOCX、Markdown：标题与段落；
- TXT：行号范围。

引用前仍应打开原文核对，因为分页、OCR 和文件版本都可能影响最终定位。

## 什么时候启用筛选

筛选适合表达真实研究边界，例如：

- 只看某一时间段；
- 只看论文、案例或法源；
- 限定某个分类；
- 排除不适合当前问题的来源。

不要为了“让结果看起来整齐”而过早加很多筛选。先看一轮全局结果，再决定收窄哪里。

## 一个可复用的检索流程

<div class="steps">

### 宽搜

用 2～4 个核心概念找到候选文献。

### 看前十条

快速检查题名、摘要、年份和来源，找出术语差异。

### 改写查询

加入新发现的同义词、制度名、法域或争议焦点。

### 深索证据

对最相关文献提出可定位的具体问题。

### 回原文核对

确认上下文、页码或章节，再记录到笔记或综述库。

</div>

## 结果不理想时

| 问题 | 调整方法 |
|---|---|
| 结果太少 | 删除次要限定，换用上位概念或同义词 |
| 结果太多 | 加入法域、制度、时间或争点 |
| 只命中标题 | 等全文处理完成，再使用深索 |
| 命中片段不准 | 把问题改成更具体的证据问题 |
| 扫描 PDF 无结果 | 先做 OCR，再更新知识库 |
| 新文件找不到 | 执行“立即更新知识库” |
