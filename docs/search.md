# 深索与全文检索

## 入库和深索的区别

> 入库让 PaperPiggy 知道“有这篇文献”；深索让它真正能够读取和检索全文。

未深索文献通常只有题录和摘要。完成深索后，系统才能更可靠地检索正文、定位原文、跨篇综合和核对引文。

不同格式的定位方式如下：

| 格式 | 原文位置 |
|---|---|
| PDF | PDF 页码，并尽量映射期刊印刷页码 |
| EPUB | 章节 |
| DOCX、Markdown | 标题和段落 |
| TXT | 行号 |

PDF 没有文字层时，PaperPiggy 会在本机 OCR，只把识别文字写入索引，不上传或修改原文件。

## 深索第一批文献

1. 打开“📚 文献”。
2. 在状态筛选中选择“未深索”。
3. 勾选当前课题最重要的一批文献。
4. 点击“深索选中”。
5. 查看顶部进度条。

深索支持断点续跑。已经完成的文献会保留，下次可以继续处理剩余部分。

### 新手应该深索多少

不要把“全库处理完”当作开始使用的前提。建议先深索当前研究方向最重要的几十篇，做几次全文检索和问答，确认效果后再扩大范围。

API 模式的嵌入以及服务端生成检索摘要可能涉及额度消耗。大批量处理前，请先查看服务商规则。

## 检索摘要

检索摘要是深索之后的第二步：

```text
① 深索：把全文拆成可检索的小段
② 检索摘要：帮助系统更容易判断整篇文献讲什么
```

打开“⚙ 设置 → 建库 → 检索摘要”，可以选择：

- **交给 Agent 生成**：推荐给已接 Agent 的用户；Agent 维护知识库时读取正文、写摘要并重嵌入。
- **服务端自动生成**：应用使用配置的 API Key 自动生成，比较省事，但可能消耗额度。
- **不生成**：仍可全文检索，只是不额外添加摘要前缀。

PaperPiggy 会做质量检查。可以在文献页筛选“摘要有效、摘要异常、摘要缺失”。

## 第一次全文检索

打开“📚 文献”，把搜索模式切换为：

```text
全文与相关内容
```

输入一个完整的研究问题，而不是只写单个关键词。例如：

```text
附条件不起诉制度在实践中存在哪些程序保障问题？
```

可以选择：

- **综合相关**：同时考虑内容相关性和来源评价；
- **内容相关**：更强调文本内容；
- **来源评价**：更强调来源层级。

## 如何判断结果是否可靠

拿到结果后检查：

1. 命中的是否是多篇不同文献；
2. 原文片段是否真正回答问题；
3. 页码、章节、段落或行号是否能够回溯；
4. 来源评价是否适合当前研究用途；
5. 重要引文是否已经打开原文再次核对。

检索结果是发现证据的入口，不应在没有核对原文的情况下直接成为最终引文。

## 常见状态

如果深索没有成功，不要把所有情况都叫“OCR失败”。文献可能是：

- 等待 OCR；
- OCR 失败；
- 附件丢失；
- 文件损坏；
- 原生文字层提取成功；
- OCR 提取成功；
- 其他全文格式读取成功。

根据具体状态处理，才能避免反复重跑无效任务。
