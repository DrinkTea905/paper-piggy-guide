## Agent 研究流程

你只需要向 Agent 说明研究目标和完成标准，由 Agent 选择知识库工具并安排执行顺序。

你负责说明：

- 想研究什么；
- 范围多大；
- 最终要什么成果；
- 证据需要严格到什么程度。

Agent 负责选择工具、安排顺序和报告中间结果。

## 怎样把任务说清楚

一个好任务通常包含四部分：

| 部分 | 你要说明什么 | 示例 |
|---|---|---|
| 研究问题 | 想解决的具体问题 | 平台算法管理如何影响劳动关系认定 |
| 范围 | 法域、年份、分类或文献数量 | 以中文法学文献为主，优先近五年 |
| 交付物 | 希望得到什么 | 观点分组、三级提纲、资料汇编 |
| 证据标准 | 怎样才算完成 | 每个实质论断给出可核对的原文位置 |

<div class="prompt">研究“自动化行政中的说明理由义务”。先检查库内覆盖情况，再把相关文献按理论基础、制度方案和反对意见分组。每组精读最有代表性的文献，给出原文位置；最后生成一份三级提纲，并把资料不足的部分单独列出。</div>

## Agent 通常会怎样执行

<div class="flow">
  <span>检查状态</span>
  <span>发现型检索</span>
  <span>定向深读</span>
  <span>核验与交付</span>
</div>

### 1. 检查知识库

Agent 先了解入库数量、深索进度、检索摘要状态和分类，避免在一个尚未准备好的范围里直接下结论。

### 2. 做发现型检索

先铺开不同文献，找出主要观点和代表来源，而不是让同一篇文献的很多相似段落占满结果。

### 3. 对关键来源定向深读

选定重要文献后，再读取原文、定位页码或章节，并补做必要的深索。

### 4. 核验论断和引文

正式交付前，Agent 可以检查一句引文是否真的出现在原文中，也可以判断一个实质论断是被库内证据支持、与证据不一致，还是本库暂时没有依据。

## 深索应该跟着研究任务走

深索是把全文拆成可检索片段的过程。大库一次跑完很慢，更实用的做法是：

1. 先围绕当前课题检索候选文献；
2. 只深索最相关的一批；
3. 为这批文献生成检索摘要；
4. 开始研究和写作；
5. 夜间空闲时再扩大范围。

<div class="prompt">不要立刻深索全库。围绕这个课题先选 10 篇最值得读的文献，分两批完成深索和检索摘要。每批结束后报告成功、失败和需要我处理的附件问题。</div>

扫描版 PDF 不需要先排除。PaperPiggy 会在深索时自动进行本地 OCR，只是处理时间会更长。

## 几类常用任务

### 找文献并建立研究地图

<div class="prompt">搜索库内关于“生成式人工智能训练数据与合理使用”的文献，按支持、反对和折中立场分组。先给研究地图，不要立刻写成文章。</div>

### 精读一篇文献

<div class="prompt">找到《文献标题》，先读取题录和深索状态；精读全文，按研究问题、方法、论证结构、主要结论和局限整理，所有关键判断给出原文位置。</div>

### 比较多篇观点

<div class="prompt">比较这 6 篇文献对平台责任边界的不同判断标准。把概念差异、规范依据和实际结论分开，不要只做摘要拼接。</div>

### 核对引文或论断

<div class="prompt">核对下面这句话是否真的出现在我的知识库原文中，并返回准确位置和上下文。如果只是近似表达，请明确说明：……</div>

### 写综述或资料汇编

<div class="prompt">按照 PaperPiggy 的“写论文与综述”工作流完成这个主题。先提交选题拆解和三级大纲，我确认后再逐节召回证据和写作。</div>

## 什么时候还要打开“文献”页

Agent 是主流程，但应用中的“文献”页仍然适合：

- 快速浏览 Zotero 分类；
- 查看某个范围已深索多少篇；
- 筛选检索摘要有效、异常或缺失的文献；
- 人工挑选一批文献开始深索；
- 打开原文件核对内容。

它更像知识库的可视化管理台，而不是你每天必须手动完成全部研究步骤的地方。
